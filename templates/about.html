About Obsy

Obsy is a complete automated observatory control system for Linux. It is reliant on the INDI protocol for telescope, mount, and observatory control and makes heavy use of KStars and EKOS for endpoint device operation. The primary role of Obsy is to provide a facility for the end user to specify the targets that require data collection, manage the collection of that data, and present the results to the user.

Pre-requisites

Before implementing Obsy the user should have a fully implemented automated observatory running under Linux and INDI. Obsy feeds schedules into EKOS so something isn't working in EKOS scheduled jobs under Obsy won't complete.

Current Status

Currently in development with no release candidate.

To Do

The following tasks are outstanding:

  • Target subsystem - Allows the user to specify targets and class them in accordance with the processing needed to create usable data. Categories are:
    • Variable stars - cadence and capture information, AAVSO integration for comparison stars to enable automated differential photometry
    • Astrophotography - capture and post-processing information
    • Occultations - capture and processing information for singular events
    • Exoplanet Transits - capture and processing information for repeating transit events
  • Schedule subsystem
  • Results subsystem - As data is collected the artifacts are cataloged by the Results subsystem and matched against schedule tasks to determine whether the task is complete. This subsystem also displays results to the user and if configured sends notifications to email, text, Twitter, etc.
  • Executive subsystem - this is a module that resides on an INDI server and does the actual execution of the job list created by the scheduler, creating XML files for the EKOS Scheduler and Camera modules and using DBUS to invoke EKOS to run the nightly schedule.

Currently in development with no release candidate.
